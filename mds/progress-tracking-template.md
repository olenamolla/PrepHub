## How to Use This Template

1. Create a new file for each month in the `progress-tracking` folder
2. Copy the daily stand-up template for each working day
3. Fill out the weekly review at the end of each week
4. Complete the monthly review at the end of each month
5. Be honest and detailed in your updates
6. Include both technical and non-technical challenges
7. Track time spent on tasks
8. Note down learning resources used

## Tips for Effective Progress Tracking

1. Update daily stand-ups first thing in the morning
2. Be specific about tasks completed and planned
3. Flag blockers immediately
4. Include links to relevant PRs or commits
5. Document important decisions
6. Track time accurately
7. Note down questions for review
8. Keep track of learning resources used

## Red Flags to Watch For

- Same blocker appearing multiple days in a row
- Large deviations from estimated time
- Repeated technical issues
- Skipped tests or documentation
- Mounting technical debt
- Scope creep
- Missing deadlines repeatedly
